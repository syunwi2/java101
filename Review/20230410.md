2. 클래스
    - 객체
        
        현실세계에 존재하는 모든 사물을 의미한다.
        
        사람이 인식할 수 있는 물체들로, 특별히 프로그래밍에서는 요구사항에 의해 추출되며 속성과 동작을 가진다.
        
    - 클래스
        
        구현 단계에서 객체의 공통의 속성과 동작으로 정의되며, 이를 각각 인스턴스 변수와 인스턴스 메서드라 한다.
        
    - 생성자
        
        따로 정의하지 않으면 기본 생성자가 자동으로 정의된다.
        
        변수의 데이터 타입, 갯수, 순서를 달리해 오버로딩이 가능하다.
        
        - this
            
            해당 인스턴스 참조값을 반환하는 변수이다.
            
3. 자바의 클래스
    
    자바는 객체 지향 프로그래밍 언어의 하나이다.
    
    자바 언어의 실행하기 위한 최소 단위는 클래스이다.
    
    클래스를 정의할 때는 아래와 같이 한다.
    
    - `public class ClassName {}`
    - `ClassName.java`
        
        ⇒ 코드 내 클래스 명과 .java 파일 명을 일치시킨다.
        
    - 인스턴스 변수, 생성자, 인스턴스 메서드를 포함한다.
    
    클래스는 메인 메서드가 없으므로 스스로 실행될 수 없으며, 핸들링 클래스가 동작을 관리한다.
    
    - 클래스 로딩(생성)
    - 생성자 혹은 직접 참조를 통한 값 저장
    - 값 조회 및 출력 등

5. new와 생성자
    - new
        
        클래스 인스턴스를 생성한다.
        
        new 뒤에 생성자를 호출한다.
        
    - 생성자
        
        클래스를 생성할 때 호출된다.
        
        클래스를 정의할 때 함께 선언하지 않으면 기본 생성자를 자동으로 호출한다.
        
        - 기본 생성자
            
            `public ClassName() {}`
            
        
        생성자를 명시적으로 정의할 경우 기본 생성자는 암묵적으로 정의되지 않는다.
        
        생성자는 여러 번 정의 가능하며 서로 다른 생성자는 서로 다른 argument list를 가진다.
        
        - argument list
            - 변수의 데이터 타입
            - 변수의 갯수
            - 변수의 순서
            
            위의 세 가지가 다르면 서로 다르다.
            
    - new와 메모리
        - 참조형 변수
            - stack에 생성된다.
        - new 키워드로 생성자 호출
            - heap에 인스턴스 변수, 인스턴스 메서드의 선언 크기만큼 메모리를 확보한다.
        - 생성자 동작
            - 파라미터(로컬 변수)에 값을 복사받는다.
            - 인스턴스 변수를 초기화한다.
            - 파라미터가 스택 프레임에서 삭제된다.
