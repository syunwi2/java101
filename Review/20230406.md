1. 자바 가상 머신
    
    자바 프로그램은 O/S에서 바로 실행되는 것이 아니라 JVM에서 실행된다.
    
    JVM은 .class 파일들을 실행할 수 있는 소프트웨어이다.
    
    JRE는 .class 파일들과 JVM을 합친 것으로 자바를 실행할 수 있는 환경이다.
    
    JDK는 자바를 실행하며 개발이 가능한 환경이다.
    
    JVM은 O/S위에 설치되므로 O/S의존적이고, 자바 프로그램은 JVM에서 실행되므로 O/S독립적이다.
    
2. 변수
    
    변수를 선언해 데이터를 저장할 수 있다.
    
    `datatype varName = data;`
    
    변수는 선언된 블록 내에서 사용 가능하며, 블록 내에서는 같은 이름의 변수를 두 번 이상 선언할 수 없다.
    
    - local variable
        
        메서드 안에 선언되며, 메서드가 호출되면 생성되고 메서드가 종료되면 소멸한다.
        
    - instance variable
        
        객체 안에 선언되며, 객체 생성 시 생성되고, 객체 소멸 시 소멸한다.
        
    - class variable
        
        프로그램 내부에 선언되며, 프로그램 실행 시 생성되고, 프로그램 종료 시 소멸한다.
        
    
    local variable은 반드시 초기화하고 사용한다.
    
    instance variable과 class variable은 데이터 타입에 따라 다음과 같이 초기화된다.
    
    - 정수형변수: 0
    - 실수형 변수: 0.0
    - **논리형 변수: false**
    - **문자형 변수: \u0000**
3. 형 변환
    1. 묵시적 형 변환
        1. 서로 다른 데이터 타입의 연산 결과는 표현 범위가 더 큰 데이터 타입이다.
        2. int형 보다 표현 범위가 작은 데이터 타입의 연산 결과는 int타입이다.
        3. Stiring과 + 연산을 한 결과는 String형이다.
        4. 서로 다른 데이터 타입의 할당 결과는 표현 범위가 더 큰 데이터 타입이다.
        5. 문자형의 연산 결과는 int형이다.
    2. 명시적 형 변환
        
        형 변환 연산자를 사용한다.
        
        `(datatype)varName`
        
4. 연산
    1. 산술 연산자
        
        가, 감, 승, 몫, 나머지
        
    2. 대입 연산자
        
        대입, 단항 연산 대입(가감승제)
        
    3. 비교 연산자
        
        같음, 다름, 부등호
        
    4. 논리 연산자
        
        그리고, 혹은, 아님
        
    5. 증감 연산자
        
        전후위 증감
        
    6. 삼항 연산자
        
        `(cond)? trueVar: falseVar;`
        
    7. 참조형 타입의 데이터 타입 체크
        
        `var instanceof Class`
